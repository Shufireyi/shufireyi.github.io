# 1.题外话 GraphQL

大家肯定听说过 GraphQL，只是应该还在用 restFul ～ 在上一个项目中，我用 restFul API 真的真的很伤心。我司做股票资讯的嘛，数据量可想而知。 后台做微服务，Fine ～ 代码很优雅～ 可我在一个页面中需要的数据，得请求 4 个 api，并且返回的数据中，利用率不足 40%！ 因为这几个 api 有依赖关系，从请求数据到页面渲染就费了很长时间～

另一个组好像已经在用 GraphQL 了，所以我也想用，哈哈哈哈～我就花了一周去学习了一下～

在 GitHub 提供的 API v4，就是 GraphQL 的，所以只想体验一下 GrahqQL 的前端可以用一下[戳这里],还可以在线体验呢[戳这里]

这次，学习小程序就写一个 GitHub 的小程序版 哈哈哈，胆子真肥～会用到部分 GraphQL，顺便简单学一下下～

# 2.小程序

{% asset_img githubsearch.jpg %} 这是已经上线小程序，可以扫一下体验一下，现在支持以下功能：

  • trending 这是 GitHub 为黑马准备的排行榜，就是每隔一段时间会更新一下有 repositories 和 developers 两个
  • search 仓库搜索，这个不需要介绍
  • 用户登录 这一块由于 wx 比较特殊不支持 oauth 方式的三方登录。我考虑一下可以直接用账户密码登录，但好像用户会觉得不安全。还有一种就是，粘贴 token。github 有提供供授权 token，在用户设置里面。这部分功能暂时没有动！
  • 用户详情，用户个人信息，以及 star 数较多的项目；如果是组织的话，就是组织信息呗～
  • 仓库详情，这部分只是将仓库的 README 解析了出来，并没有提供看代码的功能 😂

目前，完成这些，后续会对搜索进行优化～ 有兴趣可以看一下[github]，star 就不用了，我也菜
