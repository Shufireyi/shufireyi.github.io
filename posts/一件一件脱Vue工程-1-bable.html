    # step1. babel 是个啥？

    {% asset_img babel-site.jpg %} 正如官网所说，babel 是一个 javascript 编译器。啥？！javascipt 还需要编译器？不是一个脚本吗，在浏览器可以直接运行的呀。 话是这么说，但大家也肯定听说过什么 ES6，ES7，ES8 什么的。那么这些 JavaScript 新语法，浏览器还能运行吗？

    呐～，现在知道 babel 是干啥的了吧。他就是一个转译器，主要将那些 js 的新的标准（但浏览器还不能去运行）翻译成浏览器可以运行的代码。

    # 在工程中使用

    为了切题，只解释一下 vue-cli 初始化后的 babel 的使用。 下面就是用 vue init webpack 得到的项目目录了，和 babel 相关的就 .babelrc 了。 {% asset_img vue-cli.jpg %}

    它里面是这样色的～

    {
  "presets": [
    ["env", {
      "modules": false, // 是否将 es6 模块语法转译
      "targets": { // 设置转译目标，使用率 >1% 的浏览器的最新两个版本。不要ie8以下～
        "browsers": ["> 1%", "last 2 versions", "not ie <= 8"]
      }
    }],
    "stage-2"
  ],
  "plugins": ["transform-vue-jsx", "transform-runtime"]
}

    以下部分是 package.json 中和 babel 相关的部分。

    // webpack 的 loader
"babel-loader": "^7.1.1",
// babel 的主要东东
"babel-core": "^6.22.1", // 如果某些代码需要调用 Babel 的 API 进行转码，就要使用 babel-core 模块
"babel-preset-env": "^1.3.2", // 转译器
"babel-preset-stage-2": "^6.22.0" // 转译器
"babel-plugin-transform-runtime": "^6.22.0", // 一些 polyfill 和 打包优化
// jsx相关 用不到jsx语法就可以忽略
"babel-plugin-syntax-jsx": "^6.18.0",
"babel-plugin-transform-vue-jsx": "^3.5.0", // 编译 JSX
"babel-helper-vue-jsx-merge-props": "^2.0.3",
// 一个对 Babel 解析器的包装，使其能够与 ESLint 兼容。在 eslint 部分会再涉及
"babel-eslint": "^8.2.1",

    下面就结合上面两个文件，解释一下。

    {
  // 中文翻译呢就是 “预先配置的东西”
  "presets":[],
  // 中文翻译呢就是 “插件”
  "plugins":[]
}

    plugins 是转译插件，其功能单一。比如 "transform-es2015-arrow-functions" 只负责转译 => 函数。 "transform-vue-jsx" 负责转译 jsx 语法。 "transform-runtime" 会自动提供部分 polyfill，并且优化打包代码[详情请戳]。polyfill 和 runtime[对比请戳]

    presets 是转译器，它是一个集合，包含了很多转译插件。不需要手动写很多转译插件。 "babel-preset-env" 就是一个很厉害的转译器。他可以针对不同平台，不同模块进行转译，说白了就像是按需转码的意思。[官网介绍] "babel-preset-stage-2" 对 es7 的新语法已经有初步规范的语法进行转译。对于 stage-2 是啥～官网介绍

    我先说的plugins 再说的presets～其实，在 babel 工作的时候，也是先进行plugins,再进行presets操作。

    不同的是plugins操作是从前往后，而presets操作是从后往前。
